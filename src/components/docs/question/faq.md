# 如何高效反馈问题

感谢您向我们反馈问题。

1. 我们推荐如果是小问题（错别字修改，小的 bug fix）直接提交 PR。
2. 如果是一个新需求，请提供：详细需求描述，最好是有伪代码示意。
3. 如果是一个 BUG，请提供：复现步骤，错误日志，系统和浏览器版本以及相关配置。如果可以，尽可能使用 gem-mine 提供一个最小可复现的代码仓库，方便我们排查问题，不要挤牙膏似的交流。
